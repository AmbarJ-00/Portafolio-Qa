# Implementation Plan - MySQL Migration, Security, and Error System

Migrate the portfolio's persistence from `localStorage` to a MySQL database, strengthen authentication using bcrypt and JWT, enforce OWASP Top 10 security measures (CSP headers, parameterized SQL, CORS, XSS sanitization), build a visual error code system (#DB-500, #Auth-401, #Permission-403, #Config-001, #Server-500), and implement corresponding test suites.

## User Review Required

> [!IMPORTANT]
> **Database Configuration & Self-Healing Schemas**: The Express server will validate the MySQL connection on boot. If the tables do not exist, the server will automatically run migrations to create them, seed default values, and setup the default admin user (`admin` / `AdminQA#2026`).

> [!WARNING]
> **Production serving**: The Express server (`server.js`) will serve the production static files from `dist/` and expose the API on port 3001. During development, Vite (port 3000) will proxy `/api` requests to the Express server.
