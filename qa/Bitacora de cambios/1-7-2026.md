# Plan de Implementación - Integración de FormSubmit

Este plan detalla los pasos para migrar la lógica de envío de mensajes de contacto de Nodemailer a **FormSubmit**, respetando la regla de no modificar UI/estilos a menos que sea necesario para la integración y las reglas de idioma (español).

## Revisión del Usuario Requerida

> [!IMPORTANT]
> - Utilizaremos la API de FormSubmit enviando peticiones POST mediante `fetch` desde el backend o conectando directamente.
> - Evaluando si es mejor enviar la petición POST desde nuestro backend `server.js` hacia `https://formsubmit.co/ajax/your@email.com` para no exponer el correo en el frontend y mantener la misma estructura del API local `/api/contact`.

## Cambios Propuestos

### Componente: Backend / API de Contacto

#### [MODIFY] [server.js](file:///c:/Users/ambar/.gemini/antigravity-ide/scratch/qa-portfolio/server.js)
- Reemplazar la sección de Nodemailer en el endpoint `/api/contact` por una llamada `fetch` (utilizando la librería nativa de Node.js o una alternativa integrada) que envíe los datos a `https://formsubmit.co/ajax/TARGET_EMAIL`.
- De esta forma, el frontend sigue llamando a `/api/contact` sin cambios de UI, y el backend delega el envío a FormSubmit de manera segura.

## Plan de Verificación

### Pruebas Manuales y Automatizadas
- Ejecutar `node qa/contact.test.js` para asegurar que el endpoint local `/api/contact` sigue validando y respondiendo correctamente.
