# Plan de Implementación — Requisitos Adicionales Completos

## Descripción General

Este plan cubre la implementación completa de todos los requisitos adicionales solicitados: sistema de confirmaciones, auditoría de botones, estados avanzados de cards, módulo About Me, control del Hero, métricas dinámicas, manejo de errores, ampliación de Appearance, y documentación/QA completa.

La aplicación es un portafolio QA (React + Vite + Tailwind), con un panel administrativo en `/backoffice` y vistas públicas. El estado se gestiona via `PortfolioContext` con persistencia en `localStorage`.

---

## Análisis del Estado Actual

### Problemas Críticos Identificados

1. **Sin feedback post-acción**: Ningún manager muestra mensajes de confirmación ni alertas de error.
2. **Eliminaciones sin confirmación**: `deleteProject`, `deleteSkill`, `deleteCertification` etc. se ejecutan directamente sin popup.
3. **Demo URL obligatoria**: El schema de Zod valida `demo` como URL requerida — debe ser opcional.
4. **Projects.jsx usa titleKey/descriptionKey**: Los proyectos creados via admin NO tienen estas claves, por lo que `t(undefined)` retorna cadena vacía en las cards públicas.
5. **Sin módulo About Me administrativo**.
6. **Sin estados avanzados** (Mantenimiento, En proceso de adquisición).
7. **Sin página 404** ni manejo global de errores.
8. **Métricas del Hero hardcodeadas** ("230+", "94%", "15+").
9. **Sin ruta para About Me en admin**.
10. **Skills con descripción larga** pueden romper el modal.

---

## Cambios Propuestos

### COMPONENTE 1 — Sistema Global de Confirmaciones y Diálogos

#### [NEW] `src/components/Toast.jsx`
Componente de notificaciones toast reutilizable (éxito, error, info, advertencia). Animado, auto-dismiss en 4s.

#### [NEW] `src/components/ConfirmDialog.jsx`
Modal de confirmación de eliminación reutilizable con texto configurable, botón "Confirmar" y "Cancelar".

#### [NEW] `src/context/ToastContext.jsx`
Context para manejar el estado global del sistema de toasts. Provee `showToast(message, type)`.

#### [MODIFY] `src/main.jsx` o `src/App.jsx`
Envolver toda la app con `ToastProvider` para acceso global.

### COMPONENTE 2 — Corrección del Flujo de Datos Proyectos

#### [MODIFY] `src/pages/Projects.jsx`
- Reemplazar `t(project.titleKey)` por lógica dual: si tiene `titleKey`, usar traducción; si tiene `title` directo, usar ese.
- Mismo fix para `descriptionKey` vs `description`.
- Aplicar filtro de visibilidad por `status`: solo mostrar proyectos con `status === 'active'` o sin status definido (compatibilidad hacia atrás).
- Aplicar lógica de "Ver Demo" según `demoVisibility` del proyecto.

#### [MODIFY] `src/pages/ProjectDetail.jsx`
- Mismo fix de title/description dual.
- Mostrar botón Demo solo si `demoVisibility !== 'hidden'` y existe URL válida.

#### [MODIFY] `src/context/PortfolioContext.jsx`
- Agregar campos `status` y `demoVisibility` a `addProject` y `updateProject`.
- Agregar `aboutItems` array al store para el módulo About Me.
- Agregar acciones: `addAboutItem`, `updateAboutItem`, `deleteAboutItem`, `reorderAboutItems`.
- Agregar `heroCards` array al store con las cards del Hero (liderazgo, etc.).
- Agregar acciones CRUD para `heroCards`.

#### [MODIFY] `src/admin/sections/ProjectsManager.jsx`
- Corregir schema Zod: `demo` pasa a ser `z.string().url().optional().or(z.literal(''))`.
- Agregar campo `status` (select: Activo, Inactivo, En mantenimiento, En proceso de adquisición).
- Agregar campo `demoVisibility` (radio: Mostrar, Ocultar, Auto).
- Integrar Toast para confirmaciones de creación/edición.
- Reemplazar `actions.deleteProject(id)` con `ConfirmDialog`.

#### [MODIFY] `src/pages/Skills.jsx`

#### [MODIFY] `src/pages/Certifications.jsx`

- `learning`: muestra barra de progreso animada, ícono de libro/flecha de crecimiento, título "EN PROCESO DE ADQUISICIÓN", descripción predefinida.
