# Plan de Implementación — QA Portfolio: Diagnóstico Vercel + Modificaciones Pendientes

## Diagnóstico del Problema de Despliegue en Vercel

### ✅ El build local funciona (2004 módulos, sin errores de compilación)

### 🔴 Causas identificadas del fallo en Vercel

| # | Problema | Impacto | Archivo |
|---|----------|---------|---------|
| 1 | **`pnpm-workspace.yaml` incorrecto** — Solo tiene `allowBuilds: esbuild: true`, pero Vercel trata este directorio como un **monorepo** y NO ejecuta `pnpm install` ni `pnpm build` en la raíz por defecto | **CRÍTICO** — Vercel no encuentra el build output | `pnpm-workspace.yaml` |
| 2 | **`vercel.json` sin `buildCommand` ni `outputDirectory`** — Vercel necesita saber que el build output está en `dist/` y que el comando de build es `pnpm run build` | **ALTO** — Sin esto, Vercel busca en el directorio equivocado | `vercel.json` |
| 3 | **`Skills.jsx` importa `react-icons`** → `GiAlienBug` en `Home.jsx` y `Projects.jsx` — `react-icons` tiene 748KB de chunk en el build. Esto no bloquea Vercel pero puede causar timeouts en builds lentos | MEDIO | `Home.jsx`, `Projects.jsx` |
| 4 | **`Skills-BF-NYno3.js` pesa 748KB** — Vite detectó que `import * as LucideIcons` crea un bundle gigante. Esto puede causar que el deploy de Vercel falle por timeout en builds | ALTO | `Skills.jsx` |

### La solución principal para Vercel:

```json
// vercel.json corregido
{
  "buildCommand": "pnpm run build",
  "outputDirectory": "dist",
  "installCommand": "pnpm install",
  "cleanUrls": true,
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

Y el `pnpm-workspace.yaml` debe quedar vacío o eliminarse (no es un monorepo):

```yaml
# pnpm-workspace.yaml — no es un workspace monorepo, debe tener el formato correcto
packages:
  - "."
```
