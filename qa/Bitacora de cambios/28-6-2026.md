## ❌ LO QUE FALTA / ESTÁ INCOMPLETO

### 🔴 CRÍTICO 1 — Clases CSS Inconsistentes (antiguas vs nuevas)
**Todos estos archivos aún usan clases Tailwind de la paleta vieja que NO existen:**
- `Contact.jsx` → `brand-navy-*`, `brand-ash-*`, `brand-electric-*`, `brand-lilac-*`
- `About.jsx` → `brand-navy-*`
- `Home.jsx` → `brand-navy-*`, etc.
- `Certifications.jsx` → clases antiguas
- `Documentation.jsx` → clases antiguas
- `Projects.jsx` → clases antiguas
- `ProjectDetail.jsx` → clases antiguas
- `CustomModule.jsx` → clases antiguas
- `NotFound.jsx` → clases antiguas
- `Navbar.jsx` → clases antiguas
- `Footer.jsx` → clases antiguas
- `Layout.jsx` → clases antiguas
- `ErrorScreen.jsx` → clases antiguas
- `App.jsx` → clases antiguas
- `AppearanceManager.jsx` → clases antiguas
- `ModulesManager.jsx` → clases antiguas

**Impacto:** La paleta premium definida en `index.css` **no se aplica visualmente** porque los componentes usan clases que no existen. La UI no se ve como se diseñó.

**Solución requerida:** Reemplazar todas las clases `brand-navy-*`, `brand-ash-*`, `brand-electric-*`, `brand-lilac-*`, `brand-purple-*`, `brand-gold-*` por las variables CSS (`var(--color-text)`, etc.) o las nuevas clases Tailwind (`yankees-*`, `burlywood-*`, etc.).

---

### 🔴 CRÍTICO 2 — Email SMTP Sin Configurar
El `.env` tiene `SMTP_USER=` y `SMTP_PASS=` **vacíos**.

**Impacto:** Los correos **no se envían realmente**. El sistema funciona en modo "fallback silencioso" (guarda localmente pero no manda email).

**Solución requerida:** El usuario debe configurar credenciales SMTP reales (Gmail App Password u otro proveedor) en `.env` y en Vercel Dashboard.

---

### 🔴 CRÍTICO 3 — Contact.jsx Usa Clases Antiguas
Además del problema de paleta, `Contact.jsx` usa la clase `bg-gradient-to-r from-brand-navy-800 to-brand-navy-900` en el botón de envío — **esto no se renderiza correctamente**.

El formulario no usa las variables CSS del nuevo tema (ej: `var(--color-button)`, `var(--bg-global)`).
