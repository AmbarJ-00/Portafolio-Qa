# Plan de Estabilización Completa del Sistema, Migración a LocalStorage, Validación del Backoffice y Rediseño de Paleta Visual

Este plan detalla los cambios técnicos que implementaremos en la aplicación de portafolio para garantizar que toda la web y la administración funcionen completamente sin depender de una base de datos activa, utilizando `localStorage` como persistencia principal, corrigiendo el servicio de correos e implementando el rediseño visual (Light y Dark Mode) especificado por el usuario.

Si la base de datos falla o no está configurada, la aplicación continuará funcionando con los datos de `localStorage`.
> - **Servicio de correo corregido:** Se solucionará el fallback de host SMTP incorrecto (`ambarJob007@gmail.com`) en [mail.js](file:///c:/Users/ambar/.gemini/antigravity-ide/scratch/qa-portfolio/src/config/mail.js) cambiándolo a `smtp.gmail.com`.
> - **Módulo de Skills:** Se reducirá el tamaño de las tarjetas, eliminando el porcentaje de aprendizaje y simplificando el diseño.
> - **Rediseño de Paleta Visual Global:** Se redefinirán los esquemas CSS del Light Mode y Dark Mode para aplicar estrictamente la nueva paleta de colores en todos los componentes.

### 2. Persistencia en LocalStorage (Integración Front ↔ Back)

#### [MODIFY] [PortfolioContext.jsx](file:///c:/Users/ambar/.gemini/antigravity-ide/scratch/qa-portfolio/src/context/PortfolioContext.jsx)
- Modificar `loadData` para comprobar si existe información en `localStorage` antes de recurrir a la configuración estática por defecto en caso de fallo del backend.
- Agregar un `useEffect` que detecte cambios en `store` y guarde automáticamente todo el estado del portafolio en `localStorage`.
- Actualizar todas las acciones CRUD (`updatePersonal`, `updateSEO`, `updateContact`, `updateAppearance`, `saveNavbarState`, proyectos, skills, certificaciones, plantillas, módulos y items about) para que, en caso de fallar la petición de red (API/DB offline), realicen la actualización de estado local de React, lo que a su vez se persistirá en `localStorage`.
- Cuando se añada una entidad sin conexión a la base de datos, generar IDs locales únicos usando la función `createId`.

### 4. Rediseño de Paleta Visual Global (Light y Dark Mode)

#### [MODIFY] [index.css](file:///c:/Users/ambar/.gemini/antigravity-ide/scratch/qa-portfolio/src/index.css) o archivos de estilos globales
